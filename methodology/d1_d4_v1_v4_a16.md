## Level 0 — Geometry Spec (re-confirmed from definition JSON)

**Inputs**
- `q_nope[N, 16, 512]` bf16
- `q_pe[N, 16, 64]` bf16
- `ckv_cache[P*64, 512]` bf16 (paged but treated flat — page_size=64 implicit in indexing)
- `kpe_cache[P*64, 64]` bf16
- `sparse_indices[N, 2048]` int32 (with `-1` sentinels; values are global token rows)
- `sm_scale` fp32 scalar (= 1/√192)

**Outputs**
- `output[N, 16, 512]` bf16
- `lse[N, 16]` fp32 (2-base log-sum-exp)

**Operation** (per token t, per head h):
1. `logit[t,h,k] = (q_nope[t,h,:] · ckv_cache[idx_k,:] + q_pe[t,h,:] · kpe_cache[idx_k,:]) * sm_scale`
   for valid `idx_k = sparse_indices[t,k]`, k ∈ [0, 2048)
2. `output[t,h,:] = Σₖ softmax(logit[t,h,:])ₖ * ckv_cache[idx_k,:]`  (V == K_c in MLA decode)
3. `lse[t,h] = (max_k logit[t,h,k]) · log₂e + log₂(Σ_k exp(logit[t,h,k] − max))`

**Self-similar?** NO (no recursion morphism field needed).

**Combining operator** (across k): max + sum-of-exp + sum-of-(exp·V). Online softmax. SRG (associative, commutative, distributes over the rescale step) + INV (the rescale `o *= exp(m_old − m_new)` requires INV to correct prior accumulations).

## D3 — Hardware Binding (current A16 state)

**Step 2 binding:**

| Level         | Choice in A16                                             | Atom basis                |
| ---           | ---                                                        | ---                       |
| Grid          | `dim3(splits_per_token, num_tokens)`                      | AREA(t) × split(k)        |
| Block         | 512 threads (1 CTA per (split, token))                    | FUN at block level        |
| Warp          | 16 warps × 32 lanes; **1 warp per query head**            | FUN(AREA(h) → warp)       |
| Thread/Lane   | Lane handles 16-element slice of d_ckv + 2 elem of d_kpe  | FUN(REDUCE(d) → lane shfl)|
| SMEM          | 14 KB (3 KV buffers); 24 B (3 mbarriers)                  | MEA × FXP                 |
| Async         | cp.async.bulk (TMA) + mbarrier; 3-buffer prefetch         | REL × FXP                 |
| TMEM/MMA      | NOT USED                                                  | (shape M=16 < min M=64)   |
| Cross-CTA     | scratch + dedicated reduce kernel (Pass 2)                | NOT atomic                |

**Atom intersections — currently exploited:**

| Intersection            | Optimization name        | Where in kernel                                              |
| ---                     | ---                      | ---                                                          |
| SRG × FXP               | Online Algorithm         | running m/l/o + rescale loop                                 |
| SRG × SYM               | Free Parallel Tree       | `warp_reduce_sum` (butterfly shuffle) for per-head logit     |
| FUN × SRG               | Hierarchical Reduce      | per-warp logit → cross-split partial reduction (Pass 2)      |
| REL × SRG               | Combine Group            | Q@Kc + Qp@Kp summed into single logit                        |
| SRG × PRD               | Predicated Reduction     | sentinel `-1` → logit=-inf → identity for max + 0 contribution |
| REL × FXP               | Pipeline Dependency      | mbarrier between cp.async.bulk completion and consume        |
| FXP × MEA               | Iteration Bound          | total_iters known at outer-loop entry (compile-time unroll)  |
| MEA × FXP               | Pipeline Buffer Sizing   | 3-buffer prefetch chosen against compute/TMA ratio           |
| AFM × MOR               | Address Composition      | per-CTA pointers hoisted (qn_t, kc_lane_offset constants)    |

**Atom intersections — NOT exploited (optimization candidates):**

1. **SYM × AFM (Guaranteed Coalescing) on SMEM Kc read** — partial.
   - Current per-lane SMEM read pattern: lane l reads `kc[buf][i][l*16..l*16+16]` (stride-32-byte-per-lane).
   - Bank analysis: bank for byte b = (b/4) mod 32. Lane l's first 4 bytes hit bank `(l*32/4) mod 32 = (l*8) mod 32`. Lanes 0,4,8,...,28 all hit bank 0; lanes 1,5,9,... hit bank 8; etc.
   - **8-way bank conflict** per LDG.128. Per outer iter: 32 cycles of SMEM (vs 4 cycles ideal).
   - Per-CTA: 32 outer iters × ~24 wasted cycles = ~770 cycles ≈ 0.5 µs of avoidable SMEM stall.
   - **Fix candidate**: XOR swizzle on SMEM layout, or change to "interleaved per-lane" (lane l reads `kc[buf][i][l + 32*j]` for j=0..7 packed as `__nv_bfloat162`).

2. **SRG × ATO (Global Reduce) — eliminate Pass 2 reduce kernel via cross-CTA atomic merge.**
   - Current: per-CTA writes `(m_partial, l_partial, o_partial)` to scratch; Pass 2 kernel does `m_global = max m_partial; l_global = Σ l_partial · exp(m_partial − m_global); o_global = Σ o_partial · …`
   - Atomic version: each partial CTA atomically merges into `output[t,h]` via:
     - atomicMax on m_global (lock-free)
     - rescale prior (m, l, o) state by `exp(prev_m − m_global)` — needs read-modify-write
     - atomicAdd on l, o
   - Online merge across atomic-arrived partials is NON-TRIVIAL because the rescale depends on the global max; without serialization or careful versioning, races can corrupt o.
   - **Reformulation** that does work: 1-pass atomicMax on m, then 2-pass atomicAdd on (l_unscaled, o_unscaled) where the local CTA pre-scales by its own m_partial − global_m. Requires a barrier-or-second-pass to read m_global. Net: still needs two passes — likely no win over current approach.
   - Alternative: 1-warp persistent-CTA reduce (one warp per token waits on completion-flag from each split CTA; uses cluster-level shared memory). Modest gain.

3. **MOR × AFM (XOR Swizzle smem layout transform)** — companion to #1.
   - Could combine with TMA tensor map (cp.async.bulk.tensor with `CU_TENSOR_MAP_SWIZZLE_*`). Per CLAUDE.md GT-10, swizzle mode in TMA must match SMEM descriptor mode — only relevant if we couple SMEM with tcgen05 MMA descriptors, which we don't here. So we can swizzle freely on the consumer side without TMA mode coupling.

4. **PRD × FXP (Predicate Hoisting) — split outer loop into "interior" and "tail" paths.**
   - Current inner unroll has `bool valid = (k_idx < k_end) && (idx_t[k_idx] >= 0);`
   - For interior outer iters (when `kv_per_split % K_PER_ITER == 0`, which is all our workloads), the bounds check is provably true.
   - Splitting into a fast loop (no bounds check) + tail loop (with bounds) would save ~3-5 cycles per inner iter × 4 × 32 = ~500 cycles per CTA ≈ 0.3 µs. Marginal.

5. **MEA × FXP × MOR (Reduce sparse_indices LDG amplification)** — small win.
   - Each CTA's threads independently re-read `idx_t[k_curr + i]` per inner iter (4 reads × 32 outer iters × 16 warps = 2048 LDG.32 per CTA, mostly L2-cached).
   - Cooperatively pre-load `idx_t[k_start..k_end]` into SMEM once per CTA. Slice size: max 64 ints = 256 B SMEM.
   - Saves ~2000 LDG instructions per CTA. Modest cycle savings.

6. **FUN × MEA (Persistent CTAs)** — eliminate kernel launch overhead.
   - Currently 2 kernel launches (split + reduce); each launch costs ~5-10 µs.
   - Persistent kernel with cooperative grid sync would save 1 launch (~5-10 µs).
   - For our small total wall time, this is meaningful (5-10% of mean).
   - Implementation cost: high — `cudaLaunchCooperativeKernel`, `cooperative_groups::this_grid().sync()`, kernel restructure to embed Pass 2 logic.

7. **SYM × FXP × REL (Cluster + DSMEM Q sharing)** — already analyzed in T3-4, deferred.
   - Splits of the same token redundantly load Q from L1. Cluster + DSMEM would let one CTA load Q and broadcast.
   - Per-token Q load is amortized once per CTA → small per-CTA savings. Cluster setup adds non-trivial code.

8. **K_PER_ITER tuning (MEA × FXP)** — re-examine inner-loop tile size.
   - K_PER_ITER=4 currently. With re-read pattern (kc_local rebuilt twice — once for QK, once for o_acc update), K=8 might fit register budget while halving outer-iter count and mbarrier waits.
   - Risk: register spill if compiler can't see the re-read invariance.

## D4 — Architecture Specification (current A16)

**FSM Phases (current kernel):**
```
[CTA entry]
   ├── tid==0 : init mbar[0,1,2]                                 (REL setup)
   ├── tid==0 : issue cp.async.bulk batch 0 → buf 0              (FXP prologue)
   ├── tid==0 : issue cp.async.bulk batch 1 → buf 1              (FXP prologue, prefetch +1)
   └── __syncthreads()                                            (mbar visibility)
[Per-warp Q load]
   └── per-lane LDG.128 × 2 + LDG.32 from q_nope/q_pe → registers
[Inner loop, it = 0 .. total_iters-1]
   ├── tid==0 : if has_issue → issue batch (it+2) → buf (it+2)%3 (FXP prefetch +2)
   ├── all    : mbarrier_wait_parity(buf=it%3, phase[buf])       (REL: TMA done)
   ├── per-lane: SMEM → registers (kc_local[K=4][16] + kp_local) (AFM read)
   ├── compute K independent logits via warp_reduce_sum          (SRG free tree)
   ├── batched online-softmax rescale (m_new, scale_old, p_arr)  (FXP × SRG online)
   └── update o_acc += Σᵢ p_i · kc_local[i]                      (SRG × INV)
[Output write]
   ├── per-lane: STG.128 × 4 to scratch_o (vectorized fp32)      (AFM write)
   └── if lane==0: STG to scratch_m, scratch_l                   (AFM scalar write)
[Pass 2: separate kernel — reduce]
```

**Lifetime tables:**

| Variable      | Birth                  | Death              | Storage class | Size                 |
| ---           | ---                    | ---                | ---           | ---                  |
| qn_local      | post-Q-load            | end of CTA         | registers     | 16 fp32/lane         |
| qp_local      | post-Q-load            | end of CTA         | registers     | 2 fp32/lane          |
| o_acc         | CTA entry              | end of CTA         | registers     | 16 fp32/lane         |
| row_max       | CTA entry              | end of CTA         | registers     | 1 fp32/lane (warp-uniform) |
| row_sum       | CTA entry              | end of CTA         | registers     | 1 fp32/lane (warp-uniform) |
| smem_kc[3]    | CTA SMEM               | end of CTA         | shared mem    | 3×4×512 bf16 = 12 KB |
| smem_kp[3]    | CTA SMEM               | end of CTA         | shared mem    | 3×4×64 bf16 = 1.5 KB |
| mbar[3]       | tid==0 init            | end of CTA         | shared mem    | 24 B                 |
| kc_local[K]   | inner-loop iter        | end of inner iter  | registers     | 4×16 fp32/lane       |
| logits[K]     | inner-loop iter        | end of inner iter  | registers     | 4 fp32/lane          |
| p_arr[K]      | inner-loop iter        | end of inner iter  | registers     | 4 fp32/lane          |

**Address composition (Step 5):**
- `kc_lane = smem_kc[buf] + i * 512 + lane_id * 16` — composable into FMA: `(buf*4 + i)*512 + lane*16`. Currently emitted as 3-term add; compiler likely fuses.
- `output[t,h,d] = output_base + t*8192 + h*512 + d` — one IADD3.
- `scratch_o[t,s,h,d] = scratch_o_base + ((t*S+s)*16+h)*512 + d` — one IMAD chain.

**Barriers (REL):**
- `__syncthreads()` × 1 — after mbarrier init + Q LDG, before inner loop. Required: mbar must be initialized before first wait; Q register state must be local-thread-only thereafter (no inter-warp dep on Q).
- `mbarrier_wait_parity` × `total_iters` — inside inner loop. Required: TMA destination data must be visible before SMEM read. mbarrier provides correct release/acquire semantics.

## Optimization Candidates (Ranked by Expected ROI)

| #     | Candidate                                                           | Expected Δ mean | Implementation effort |
|-------|---------------------------------------------------------------------|-----------------|------------------------|
| **O1**| **Cooperatively pre-load `sparse_indices` slice into SMEM**         | -1 to -2%       | Low (Tier 1)           |
| **O2**| **SMEM bank-conflict mitigation for per-lane Kc read** (interleaved layout, `__nv_bfloat162`-paired) | -1 to -3%       | Moderate (Tier 4)      |
| **O3**| **PRD × FXP path specialization (no bounds-check fast loop + tail)**| -0.5 to -1%     | Low (Tier 1)           |
| **O4**| **Persistent / cooperative-grid kernel: fuse split + reduce**       | -5 to -10%      | High (Tier 3)          |
| **O5**| **K_PER_ITER=8 with re-read pattern**                                | -1 to -3% or regression  | Moderate (Tier 4)       |
| **O6**| **Cluster + DSMEM Q sharing** (T3-4)                                 | -0.5 to -1%     | High (Tier 3)          |
| **O7**| **SRG × ATO atomic-based cross-split merge**                         | -0.5 to -1%     | High (Tier 3)          |

**Why we pursue O1, O2, O3 first:** they target real over-counts (idx LDG amplification, bank conflict cycles, hoist-able branches) without architectural restructure.

**Why O4 is highest-ceiling but deferred:** cooperative grid sync requires non-trivial setup; the gain is mostly launch-overhead — meaningful for our small wall time but high implementation risk.

**Why O5 is uncertain:** doubling K_PER_ITER doubles the kc_local register footprint per iter; the compiler may spill if it can't reuse the per-i scratch. Re-read mitigation requires manual restructure.

**Why O6/O7 stay deferred:** earlier T3-4/T3-7 analysis showed Q-load amortization is already small per-CTA, and atomic-based reduce can't easily handle the rescale dependency without serialization.

## Recommendation

Attempt in this order (single optimization at a time, mean-strict-decrease accept):

1. **A17 (O1)** — pre-load sparse_indices to SMEM. Cheapest, low risk, measurable.
2. **A18 (O2)** — SMEM bank-conflict layout fix. Moderate effort, modest gain.
3. **A19 (O3)** — predicate hoisting / path specialization. If gain is sub-noise, accept anyway (clean code).
4. **Re-evaluate**: if O1-O3 all rejected, the kernel is at a hard limit set by HBM-gather bandwidth contention. Stop and document.
